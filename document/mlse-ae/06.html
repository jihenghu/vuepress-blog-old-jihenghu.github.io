思考一下：

  • 为何采用MYD06？具有什么主要优点？
  • MYD06 云产品有哪些？都是什么波段探测获取的？
  • MODIS的扫描方式？

​ 我们采用MYD06的原因是其搭载于Aqua卫星上，和AMSR_E探测器处于同一平台，具有优秀的同步性，虽然分辨率不同，但是时间和空间的协调性堪称完美，我们是使用MYD06的云观测来提供云信息。大部分云参数反演是在可见光条件下进行的如Re，COT。所以夜间无反演。

# - MYD06 Level 2 数据

MODIS二级产品的分辨率有两个，5km和1km，并且由于MODIS采用的是滚动扫描（巡航方向看，顺时针滚动），其地表入射角和距离星下点的距离有关，星下点的分辨率最高，边缘则较为低。

对于1km分辨率参数来说：

resolution X Cell_Across_Swath_1km = 1354 km
1

而实际上，MODOS的扫描幅宽为 2330 km，所以轨道边缘的分辨率远远大于1 km。这个宽度足以覆盖AMSR-E的扫描幅宽（1445 km），提供足够的云信息。相对来说，AMSR-E由于采用圆锥扫描方式，地表入射角固定，分辨率自然固定，像斑大小也是固定的。

对于两个分辨率的单片段几何参数（5分钟一个片段）：

res1km 5分钟 2030次扫描，单次扫描产生1354个样本
res5km 5分钟 406次扫描，单次扫描产生270个样本

# - 云平均

因为其他云参数均有5km分辨率，但是cwp只有1km，需要对云水进行格点平均。注意，这一步分仅仅是MYD06参数内部的cwp平均，和后面collocation部分的像斑内部云平均是两回事，这里只是为了统一MYD06分辨率到5km

  • 坐标：

    MOD06/MYD06自带5km分辨率的Geolocation field，但是1km的lon2d和lat2d经纬度数据需要匹配下载MOD03/MYD03 的Geolocation 数据。但是，由于1km格点和5km格点严格遵守嵌套顺序，即一个5 km个点内部嵌套着25个1 km格点。所以不需要MYD03资料，就可以进行格点平均操作。

  • 现在，先利用 5km的经纬度信息 Latitude,Longitude计算需要裁减的5km网格四至点jStrt,jLast,iStrt,iLast；

    ;;;  glat2d,glon2d
       Latitude = hdfile[:]->Latitude  ;;; MYD06 5km geolocation [2030 * 1354] 
       Longitude = hdfile[:]->Longitude 

     ji = region_ind(Latitude,Longitude,latS,latN,lonW,lonE)
            jStrt = ji(0)      ; scan start
            jLast = ji(1)      ; scan last
            iStrt = ji(2)      ; pix start
            iLast = ji(3)      ; pix last

  • 利用jStrt,jLast,iStrt,iLast确定1km 格点下标范围jS1km,jL1km,iS1km,iL1km。

    5km网格在1km网格上的采样规律为[3, 2028, 5] x [3, 1348, 5] =[ 406 * 270 ]，即，粗网格以细网格的第[3,8,13····] * [3,8,13·····]为中心点，5个网格距采样一次，所以，可以确定出1km网格的四至点：

    ;;; do sampling [3, 2028, 5] x [3, 1348, 5]   =[ 406 * 270 ]	 	
;;; convert geolocation to resolution 1km  
            jS1km = (jStrt)*5          ; 1kmlat start
            jL1km = (jLast+1)*5-1      ; 1kmlat last
            iS1km = (iStrt)*5          ; 1kmlon start
            iL1km = (iLast+1)*5-1      ; 1kmlon last 
 ;;Extract cwp 1km in ROI

# - 数据说明

  • 对于MYD06的处理，最终输出的数据特征有：

    • 只保留了有云样本，晴空样本和缺测样本都舍弃了。如今看来，这样做有些不妥，在后面collocation时有可能把缺测的云当成晴空，思考一下，虽然有CPI来确定云是否缺测。

    • 由于MYD06原始文件名不包含上行下行轨道标志，所以处理的轨道较多，后期和AMSR-E匹配时使用扫描时间来唯一确定。

    • 输出参数包含样本序号，经纬度，扫描时间，云参数等，详见表1。

      表1. 输出参数详情

      参数 ip lon lat scanTime CWP5km

      有效范围 自然数 70-145 10-55 UTC时间（单位小时） g/m2
      输出类型 I6 F12.4 F12.4 F12.4 F12.4
      参数 CFR CPI CTP CTH CTT
      说明 云分数 相态 云顶气压 云顶高度 云顶温度
